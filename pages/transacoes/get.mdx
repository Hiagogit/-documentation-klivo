# Consultar Transação

Retorna os detalhes completos de uma transação específica usando seu hash único.

## Endpoint

```
GET https://api.klivopay.com.br/api/public/v1/transactions/:hash?api_token=SEU_TOKEN
```

## Parâmetros de URL

<ParamField path="hash" type="string" required>
  Hash único da transação
</ParamField>

## Parâmetros de Query

<ParamField query="api_token" type="string" required>
  Token de autenticação da API
</ParamField>

## Exemplos de Requisição

<CodeGroup>
```bash cURL
curl -X GET "https://api.klivopay.com.br/api/public/v1/transactions/abc123xyz?api_token=SEU_TOKEN"
```

```javascript JavaScript
const getTransaction = async (hash) => {
  const response = await fetch(
    `https://api.klivopay.com.br/api/public/v1/transactions/${hash}?api_token=SEU_TOKEN`
  );

  return await response.json();
};

// Uso
const transaction = await getTransaction('abc123xyz');
console.log(transaction);
```

```python Python
import requests

def get_transaction(hash):
    url = f"https://api.klivopay.com.br/api/public/v1/transactions/{hash}"
    params = {"api_token": "SEU_TOKEN"}

    response = requests.get(url, params=params)
    return response.json()

# Uso
transaction = get_transaction('abc123xyz')
print(transaction)
```

```php PHP
<?php

function getTransaction($hash, $apiToken) {
    $url = "https://api.klivopay.com.br/api/public/v1/transactions/{$hash}";
    $url .= "?api_token={$apiToken}";

    $ch = curl_init($url);
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

    $response = curl_exec($ch);
    curl_close($ch);

    return json_decode($response, true);
}

$transaction = getTransaction('abc123xyz', 'SEU_TOKEN');
```
</CodeGroup>

## Códigos de Status HTTP

| Código | Descrição |
|--------|-----------|
| 200 | Transação encontrada |
| 401 | Token inválido ou ausente |
| 404 | Transação não encontrada |
| 500 | Erro interno do servidor |

## Erros

### 404 - Transação Não Encontrada

```json
{
  "success": false,
  "error": "Transaction not found",
  "code": "TRANSACTION_NOT_FOUND"
}
```

## Status Possíveis

| Status | Descrição | Próxima Ação |
|--------|-----------|--------------|
| `pending` | Aguardando pagamento | Aguardar webhook ou consultar periodicamente |
| `paid` | Pagamento confirmado | **Liberar acesso ao produto** |
| `failed` | Pagamento falhou | Solicitar novo método de pagamento |
| `refunded` | Reembolsado | **Revogar acesso** |
| `canceled` | Cancelado manualmente | Registrar cancelamento |
| `expired` | Expirado (PIX) | Criar nova transação |

## Casos de Uso

### Verificar Status Após Webhook

Ao receber um webhook, confirme o status consultando a API:

```javascript
app.post('/webhook/klivopay', async (req, res) => {
  const { transaction_hash, status } = req.body;

  // Confirmar com a API
  const transaction = await getTransaction(transaction_hash);

  if (transaction.data.status !== status) {
    console.error('Status mismatch!');
    return res.status(400).send('Invalid webhook');
  }

  // Processar...
  if (transaction.data.status === 'paid') {
    await grantAccess(transaction.data.customer.email);
  }

  res.status(200).send('OK');
});
```

### Polling para Pagamento PIX

Para transações PIX sem webhook, você pode fazer polling:

```javascript
async function waitForPixPayment(transactionHash, maxAttempts = 30) {
  for (let i = 0; i < maxAttempts; i++) {
    const transaction = await getTransaction(transactionHash);

    if (transaction.data.status === 'paid') {
      return transaction.data;
    }

    if (transaction.data.status === 'expired' || transaction.data.status === 'failed') {
      throw new Error(`Payment ${transaction.data.status}`);
    }

    // Aguardar 10 segundos
    await new Promise(resolve => setTimeout(resolve, 10000));
  }

  throw new Error('Timeout waiting for payment');
}
```

<Warning>
  **Não recomendado**: Polling constante pode exceder limites de taxa. Prefira usar webhooks.
</Warning>

### Exibir Detalhes para o Cliente

```javascript
async function showTransactionDetails(hash) {
  const { data } = await getTransaction(hash);

  return {
    orderId: data.hash,
    status: translateStatus(data.status),
    amount: `R$ ${(data.amount / 100).toFixed(2)}`,
    paymentMethod: data.payment_method === 'pix' ? 'PIX' : 'Cartão de Crédito',
    installments: data.installments > 1 ? `${data.installments}x` : 'À vista',
    date: new Date(data.created_at).toLocaleDateString('pt-BR')
  };
}

function translateStatus(status) {
  const statusMap = {
    pending: 'Aguardando Pagamento',
    paid: 'Pago',
    failed: 'Falhou',
    refunded: 'Reembolsado',
    canceled: 'Cancelado',
    expired: 'Expirado'
  };

  return statusMap[status] || status;
}
```

### Buscar Detalhes do QR Code PIX

```javascript
async function getPixQrCode(transactionHash) {
  const { data } = await getTransaction(transactionHash);

  if (data.payment_method !== 'pix') {
    throw new Error('Transaction is not PIX');
  }

  if (data.status !== 'pending') {
    throw new Error('PIX already paid or expired');
  }

  return {
    qrCode: data.pix.qr_code,
    qrCodeUrl: data.pix.qr_code_url,
    expiresAt: data.pix.expires_at
  };
}
```

## Observações

<Info>
  Consultar uma transação **não altera** seu status. Use este endpoint para verificar o estado atual.
</Info>

<Note>
  Para transações PIX, o objeto `pix` contém o QR Code mesmo após o pagamento, permitindo referência histórica.
</Note>

<Tip>
  Combine este endpoint com webhooks: use webhooks para notificações em tempo real e consulte a API para confirmação.
</Tip>
