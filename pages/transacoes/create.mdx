# Criar Transação

Processa um novo pagamento através da Klivopay. Suporta pagamentos via **PIX** e **Cartão de Crédito**.

## Endpoint

```
POST https://api.klivopay.com.br/api/public/v1/transactions?api_token=SEU_TOKEN
```

## Parâmetros do Body

<ParamField body="amount" type="integer" required>
  Valor total da transação em centavos (ex: 29700 = R$ 297,00)
</ParamField>

<ParamField body="payment_method" type="string" required>
  Método de pagamento: `"pix"` ou `"credit_card"`
</ParamField>

<ParamField body="offer_hash" type="string">
  Hash da oferta associada ao produto
</ParamField>

<ParamField body="installments" type="integer" default="1">
  Número de parcelas (1-12). Aplicável apenas para cartão de crédito.
</ParamField>

<ParamField body="expire_in_days" type="integer" default="1">
  Dias para expirar (apenas para PIX, máximo 30 dias)
</ParamField>

<ParamField body="transaction_origin" type="string" default="api">
  Origem da transação: `"api"`, `"checkout"`, etc.
</ParamField>

<ParamField body="postback_url" type="string">
  URL para receber webhooks de atualização de status
</ParamField>

### Objeto card (obrigatório para credit_card)

<ParamField body="card.number" type="string" required>
  Número do cartão (sem espaços)
</ParamField>

<ParamField body="card.holder_name" type="string" required>
  Nome do titular do cartão (como impresso no cartão)
</ParamField>

<ParamField body="card.exp_month" type="integer" required>
  Mês de expiração (1-12)
</ParamField>

<ParamField body="card.exp_year" type="integer" required>
  Ano de expiração (YYYY)
</ParamField>

<ParamField body="card.cvv" type="string" required>
  Código de segurança (3-4 dígitos)
</ParamField>

### Objeto customer (obrigatório)

<ParamField body="customer.name" type="string" required>
  Nome completo do cliente
</ParamField>

<ParamField body="customer.email" type="string" required>
  Email do cliente
</ParamField>

<ParamField body="customer.phone_number" type="string" required>
  Telefone com DDD (apenas números)
</ParamField>

<ParamField body="customer.document" type="string" required>
  CPF ou CNPJ (apenas números)
</ParamField>

<ParamField body="customer.street_name" type="string">
  Nome da rua (obrigatório para cartão)
</ParamField>

<ParamField body="customer.number" type="string">
  Número do endereço (obrigatório para cartão)
</ParamField>

<ParamField body="customer.complement" type="string">
  Complemento do endereço
</ParamField>

<ParamField body="customer.neighborhood" type="string">
  Bairro (obrigatório para cartão)
</ParamField>

<ParamField body="customer.city" type="string">
  Cidade (obrigatório para cartão)
</ParamField>

<ParamField body="customer.state" type="string">
  Estado - UF (obrigatório para cartão)
</ParamField>

<ParamField body="customer.zip_code" type="string">
  CEP (apenas números, obrigatório para cartão)
</ParamField>

### Array cart (obrigatório)

<ParamField body="cart" type="array" required>
  Lista de itens do carrinho

  <Expandable title="Estrutura do item">
    <ParamField body="product_hash" type="string" required>
      Hash do produto
    </ParamField>

    <ParamField body="title" type="string" required>
      Título do produto
    </ParamField>

    <ParamField body="price" type="integer" required>
      Preço unitário em centavos
    </ParamField>

    <ParamField body="quantity" type="integer" required>
      Quantidade
    </ParamField>

    <ParamField body="operation_type" type="integer" default="1">
      Tipo de operação (1 = Venda)
    </ParamField>

    <ParamField body="tangible" type="boolean" default="false">
      Se é produto físico (true) ou digital (false)
    </ParamField>

    <ParamField body="cover" type="string">
      URL da imagem do produto
    </ParamField>

    <ParamField body="product_id" type="integer">
      ID interno do produto
    </ParamField>

    <ParamField body="offer_id" type="integer">
      ID interno da oferta
    </ParamField>
  </Expandable>
</ParamField>

## Códigos de Status HTTP

| Código | Descrição |
|--------|-----------|
| 201 | Transação criada com sucesso |
| 400 | Dados inválidos na requisição |
| 401 | Token de API inválido ou ausente |
| 422 | Erro de validação (ex: cartão recusado) |
| 500 | Erro interno do servidor |

## Erros Comuns

<AccordionGroup>
  <Accordion title="400 - Dados Inválidos">
    ```json
    {
      "success": false,
      "error": "Invalid request data",
      "details": {
        "customer.email": "Invalid email format",
        "amount": "Amount must be greater than 500"
      }
    }
    ```

    **Solução**: Verifique os dados enviados e corrija os campos indicados.
  </Accordion>

  <Accordion title="422 - Cartão Recusado">
    ```json
    {
      "success": false,
      "error": "Card declined",
      "code": "CARD_DECLINED",
      "message": "Insufficient funds"
    }
    ```

    **Solução**: Solicite ao cliente outro método de pagamento.
  </Accordion>

  <Accordion title="422 - Endereço Incompleto">
    ```json
    {
      "success": false,
      "error": "Validation failed",
      "details": {
        "customer.street_name": "Required for credit card",
        "customer.zip_code": "Required for credit card"
      }
    }
    ```

    **Solução**: Para cartão de crédito, todos os dados de endereço são obrigatórios.
  </Accordion>
</AccordionGroup>

## Observações Importantes

<Warning>
  **Segurança**: Nunca armazene dados de cartão em seu sistema. Envie diretamente para a API Klivopay.
</Warning>

<Info>
  **Webhook**: Sempre configure `postback_url` para receber atualizações automáticas de status.
</Info>

<Note>
  **PIX**: O QR Code expira no prazo definido em `expire_in_days`. Após expirar, o status muda para `expired`.
</Note>

<Tip>
  **Parcelamento**: Use o endpoint [Calcular Parcelamento](/pages/parcelamento/calculate) para obter os valores exatos com juros antes de criar a transação.
</Tip>
