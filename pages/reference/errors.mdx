# Códigos de Erro

Esta página lista todos os códigos de erro específicos retornados pela API Klivopay.

## Estrutura de Erro

Todos os erros seguem este formato:

```json
{
  "success": false,
  "error": "Mensagem de erro legível",
  "code": "CODIGO_DO_ERRO",
  "details": {
    "campo": "descrição do problema"
  }
}
```

## Erros de Autenticação

| Código | Mensagem | Causa |
|--------|----------|-------|
| `MISSING_TOKEN` | API token is required | Token não foi enviado |
| `INVALID_TOKEN` | Invalid API token | Token inválido ou expirado |
| `FORBIDDEN` | Access denied to this resource | Sem permissão para o recurso |

### Exemplos

```json
{
  "success": false,
  "error": "API token is required",
  "code": "MISSING_TOKEN"
}
```

## Erros de Validação

| Código | Mensagem | Causa |
|--------|----------|-------|
| `VALIDATION_FAILED` | Validation failed | Dados inválidos |
| `INVALID_AMOUNT` | Invalid amount | Valor inválido ou fora dos limites |
| `INVALID_EMAIL` | Invalid email format | Email mal formatado |
| `INVALID_DOCUMENT` | Invalid document number | CPF/CNPJ inválido |
| `INVALID_CATEGORY` | Invalid category | ID de categoria não existe |

### Exemplos

```json
{
  "success": false,
  "error": "Validation failed",
  "code": "VALIDATION_FAILED",
  "details": {
    "customer.email": "Invalid email format",
    "amount": "Amount must be greater than 100",
    "customer.document": "Invalid CPF"
  }
}
```

## Erros de Transação

| Código | Mensagem | Causa |
|--------|----------|-------|
| `TRANSACTION_NOT_FOUND` | Transaction not found | Hash de transação não existe |
| `TRANSACTION_NOT_PAID` | Transaction not paid | Transação não está paga |
| `CARD_DECLINED` | Card declined | Cartão recusado |
| `INSUFFICIENT_FUNDS` | Insufficient funds | Saldo insuficiente |
| `INVALID_CARD` | Invalid card | Dados do cartão inválidos |
| `EXPIRED_CARD` | Card expired | Cartão expirado |

### Exemplos

```json
{
  "success": false,
  "error": "Card declined",
  "code": "CARD_DECLINED",
  "message": "Insufficient funds"
}
```

## Erros de Reembolso

| Código | Mensagem | Causa |
|--------|----------|-------|
| `REFUND_PERIOD_EXPIRED` | Refund period expired | Passou do prazo de reembolso (180 dias) |
| `ALREADY_REFUNDED` | Transaction already refunded | Transação já foi totalmente reembolsada |
| `INVALID_REFUND_AMOUNT` | Invalid refund amount | Valor de reembolso maior que o pago |
| `REFUND_NOT_ALLOWED` | Refund not allowed | PIX ou transação não elegível |

### Exemplos

```json
{
  "success": false,
  "error": "Refund period expired",
  "code": "REFUND_PERIOD_EXPIRED",
  "message": "Refunds are only allowed within 180 days"
}
```

## Erros de Rate Limiting

| Código | Mensagem | Causa |
|--------|----------|-------|
| `RATE_LIMIT_EXCEEDED` | Rate limit exceeded | Excedeu limite de requisições |

### Exemplos

```json
{
  "success": false,
  "error": "Rate limit exceeded",
  "code": "RATE_LIMIT_EXCEEDED",
  "retry_after": 60
}
```

## Erros do Servidor

| Código | Mensagem | Causa |
|--------|----------|-------|
| `INTERNAL_ERROR` | Internal server error | Erro interno no servidor |
| `SERVICE_UNAVAILABLE` | Service temporarily unavailable | Serviço em manutenção |

### Exemplos

```json
{
  "success": false,
  "error": "Internal server error",
  "code": "INTERNAL_ERROR"
}
```

## Tratamento Recomendado

### Por Código HTTP

```javascript
const handleError = (response, errorData) => {
  switch (response.status) {
    case 400:
      // Dados inválidos - mostrar ao usuário
      showValidationErrors(errorData.details);
      break;

    case 401:
      // Token inválido - renovar ou logout
      redirectToLogin();
      break;

    case 404:
      // Recurso não encontrado
      showNotFoundMessage();
      break;

    case 422:
      // Erro de negócio (ex: cartão recusado)
      showBusinessError(errorData.message);
      break;

    case 429:
      // Rate limit - aguardar
      const retryAfter = errorData.retry_after || 60;
      setTimeout(() => retry(), retryAfter * 1000);
      break;

    case 500:
      // Erro do servidor - retry
      retryWithBackoff();
      break;

    default:
      showGenericError();
  }
};
```

### Por Código de Erro

```javascript
const handleErrorCode = (errorCode, errorData) => {
  switch (errorCode) {
    case 'CARD_DECLINED':
      alert('Cartão recusado. Tente outro método de pagamento.');
      break;

    case 'VALIDATION_FAILED':
      showValidationErrors(errorData.details);
      break;

    case 'REFUND_PERIOD_EXPIRED':
      alert('Prazo de reembolso expirado (180 dias).');
      break;

    case 'RATE_LIMIT_EXCEEDED':
      alert('Muitas requisições. Aguarde um momento.');
      break;

    default:
      alert(`Erro: ${errorData.error}`);
  }
};
```

## Mensagens Amigáveis

Traduza códigos de erro para mensagens amigáveis:

```javascript
const errorMessages = {
  CARD_DECLINED: 'Seu cartão foi recusado. Verifique os dados ou tente outro cartão.',
  INSUFFICIENT_FUNDS: 'Saldo insuficiente no cartão.',
  EXPIRED_CARD: 'Cartão expirado. Use outro cartão.',
  INVALID_CARD: 'Dados do cartão inválidos. Verifique as informações.',
  TRANSACTION_NOT_FOUND: 'Transação não encontrada.',
  PRODUCT_NOT_FOUND: 'Produto não encontrado.',
  REFUND_PERIOD_EXPIRED: 'O prazo para reembolso expirou.',
  INVALID_TOKEN: 'Sessão expirada. Faça login novamente.',
  RATE_LIMIT_EXCEEDED: 'Você está fazendo muitas requisições. Aguarde um momento.'
};

const getFriendlyMessage = (code) => {
  return errorMessages[code] || 'Ocorreu um erro. Tente novamente.';
};
```

## Logging de Erros

Sempre registre erros para análise:

```javascript
const logError = (error, context) => {
  console.error('API Error:', {
    code: error.code,
    message: error.error,
    status: context.status,
    endpoint: context.endpoint,
    timestamp: new Date().toISOString(),
    details: error.details
  });

  // Enviar para serviço de monitoramento
  if (error.code === 'INTERNAL_ERROR' || context.status >= 500) {
    sendToErrorTracking(error, context);
  }
};
```
