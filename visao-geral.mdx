## Conceitos Fundamentais

### Transações

Representam operações financeiras como pagamentos, reembolsos e consultas. Cada transação possui um identificador único (hash) e passa por diferentes status durante seu ciclo de vida.

<Note>
  **Exemplo:** `POST /transactions` para criar um novo pagamento
</Note>

### Produtos e Ofertas

Produtos são itens que você vende, enquanto ofertas são configurações específicas de preço e condições para esses produtos. Um produto pode ter múltiplas ofertas.

<Note>
  **Exemplo:** `GET /products` para listar seus produtos
</Note>

### Saldo e Saques

Consulte seu saldo disponível e solicite transferências para sua conta bancária. O sistema processa saques automaticamente respeitando os prazos estabelecidos.

<Note>
  **Exemplo:** `GET /balance` para consultar saldo atual
</Note>

### Contas Bancárias

Gerencie suas informações bancárias para recebimento de pagamentos. Suporte para contas correntes, poupança e chaves PIX.

<Note>
  **Exemplo:** `POST /bank-accounts` para cadastrar conta
</Note>

## HTTP e Segurança

### Comunicação Criptografada

Todas as requisições utilizam TLS 1.3 com criptografia de ponta a ponta. Headers de segurança obrigatórios e validação rigorosa de dados garantem máxima proteção em todas as comunicações.

<CardGroup cols={3}>

  <Card title="Formato de Dados" icon="code">

## Configuração de Postback

Configure uma URL em sua aplicação para receber notificações automáticas sobre mudanças no status das transações.

**Exemplo de payload recebido:**

```json
{
  "transaction_hash": "abc123def456",
  "status": "paid",
  "amount": 15000,
  "payment_method": "pix",
  "paid_at": "2025-01-20T10:15:00Z"
}
```
